# Introduction to OO Design

## Tutorial 4 – Class diagram

**Aim**: the aim of this tutorial is to ensure that the student understands how to create conceptual level class diagrams

## Exercise 1 

An exhibiting organisation keeps information about paintings and sculptures. Each painting has a painting-name, a painter-name and a painting-description. Each sculpture has a sculpture name, a sculptor-name  and a sculpture-description. Painting and sculptures may appear in the same gallery. For the purpose of keeping track of the location of the items, each painting and sculpture is given a unique identifier art-no.

Each Gallery has an identifier gallery-no and a size gallery-size. Each gallery can store any number of art objects. Each art object appears in one gallery only. The date-placed-in-gallery is kept for both paintings and sculptures. 

Customers are welcome to view the exhibition at any time. On arrival each customer is issued with a number. If they wish to purchase an item they can pay a deposit on it and the receptionist records order details including their number and the unique number of the item they wish to purchase. Items can be collected once the amount due has been paid in full. The Gallery arranges to have items shipped to the customers address should they wish but a charge is made for this service. The charge varies depending on the size of the item.

Create a conceptual level class diagram for the system as described above.
